### 1. Create Ledger Sidebar (`src/components/ledger/sidebar.tsx`)

Build a persistent left sidebar for the Ledger demo. This sidebar is visible on all 7 Ledger screens.

**Design direction:** Enterprise SaaS — clean, professional, similar to Linear or Notion's sidebar. Uses the Ledger amber theme automatically via CSS variable inheritance.

**Structure:**
```
┌─────────────────┐
│ Ledger (logo)   │
│                 │
│ ── Navigation ──│
│ 📊 Dashboard    │ ← active state with amber accent
│ 📁 Projects     │
│ 📄 Documents    │
│ ✏️ Changes      │ (don't actually use emoji — use lucide-react icons)
│ 📋 Tasks        │
│ ⚡ Conflicts    │
│ 📜 Audit Log    │
│                 │
│ ── Team ────────│
│ 👤 Sarah Chen   │
│ 👤 Marcus...    │ (show first 4 team members with online status dot)
│ +2 more         │
│                 │
│ ── bottom ──────│
│ Current project │
│ selector area   │
└─────────────────┘
```

**Requirements:**
- Width: 240px fixed
- Uses `ScrollArea` from shadcn for overflow
- Navigation items link to Ledger sub-routes:
  - Dashboard → `/app/ledger`
  - Projects → `/app/ledger/project`
  - Documents → `/app/ledger/document`
  - Changes → `/app/ledger/changes`
  - Tasks → `/app/ledger/tasks`
  - Conflicts → `/app/ledger/conflicts`
  - Audit Log → `/app/ledger/audit`
- Active state: highlight current route with amber/primary background
- Use `usePathname()` from `next/navigation` to detect active route
- lucide-react icons for each nav item: LayoutDashboard, FolderOpen, FileText, GitCompare, CheckSquare, AlertTriangle, ScrollText
- Team section: show users from mock data (import from `@/data/ledger`), with colored online/away/offline dots
- Bottom: show current project name from mock data
- Use semantic `<nav>` and `<aside>` elements
- Keyboard navigable
