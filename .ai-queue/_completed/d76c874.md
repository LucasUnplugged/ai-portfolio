## Instructions

### Step 1: Create DemoHeader

Create `src/components/demo-header.tsx`:

- A navigation bar that sits at the top of each demo app
- Left side: "← Back to Portfolio" link (links to `/`)
- Right side: Cross-demo navigation links — "Ledger", "Ritual", "Circles" linking to `/app/ledger`, `/app/ritual`, `/app/circles`
- The current demo should be visually distinguished (bold, different color, or underlined)
- Accept a `current` prop to indicate which demo is active: `"ledger" | "ritual" | "circles"`
- This component renders INSIDE each demo's themed wrapper, so it inherits the demo's CSS variables automatically
- Use `next/link` for all navigation
- Use semantic HTML: `<header>`, `<nav>`
- Keep it compact — this is secondary navigation inside a demo

```tsx
import Link from "next/link";

const demos = [
  { id: "ledger", name: "Ledger", href: "/app/ledger" },
  { id: "ritual", name: "Ritual", href: "/app/ritual" },
  { id: "circles", name: "Circles", href: "/app/circles" },
] as const;

type DemoId = (typeof demos)[number]["id"];

interface DemoHeaderProps {
  current: DemoId;
}

export function DemoHeader({ current }: DemoHeaderProps) {
  return (
    <header className="flex items-center justify-between border-b border-border px-4 py-2">
      <Link
        href="/"
        className="text-sm text-muted-foreground transition-colors hover:text-foreground"
      >
        ← Portfolio
      </Link>
      <nav className="flex items-center gap-1">
        {demos.map((demo) => (
          <Link
            key={demo.id}
            href={demo.href}
            className={`px-2 py-1 text-sm transition-colors ${
              demo.id === current
                ? "font-medium text-foreground"
                : "text-muted-foreground hover:text-foreground"
            }`}
          >
            {demo.name}
          </Link>
        ))}
      </nav>
    </header>
  );
}
```

### Step 2: Create PhoneSimulator

Create `src/components/phone-simulator.tsx`:

- A visual container that simulates a phone screen (375×812 viewport)
- Centered on the page
- Scrollable content area inside the phone frame
- Visual phone frame with rounded corners, subtle border/shadow
- The phone content area should scroll independently
- Accept `children` prop for the demo content
- This is used by Ritual (dark) and Circles (light) demos — NOT Ledger (desktop-only)

```tsx
import type { ReactNode } from "react";

interface PhoneSimulatorProps {
  children: ReactNode;
}

export function PhoneSimulator({ children }: PhoneSimulatorProps) {
  return (
    <div className="flex min-h-screen items-center justify-center p-8">
      <div className="relative w-[375px] h-[812px] rounded-[3rem] border-[8px] border-foreground/10 bg-background shadow-2xl overflow-hidden">
        {/* Status bar notch */}
        <div className="absolute top-0 left-1/2 -translate-x-1/2 w-[120px] h-[28px] bg-foreground/10 rounded-b-2xl z-10" />
        {/* Scrollable content area */}
        <div className="h-full overflow-y-auto pt-8">
          {children}
        </div>
      </div>
    </div>
  );
}
```
