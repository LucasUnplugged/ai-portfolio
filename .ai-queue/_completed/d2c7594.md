**Implementation:**

1. **Header:** Back link to Dashboard. Title "Task Board".

2. **Column layout:** 4 equal-width columns using a CSS grid (`grid-cols-4`). Each column has:
   - Column header: status name + task count in parentheses
   - Column header styling: subtle background color per status
   - Scrollable card area below header

3. **Task cards:** Use the `TaskCard` component from `@/components/ledger/task-card`. Pass task and resolved assignee.

4. **Column styling:**
   - Backlog: neutral/gray header
   - In Progress: blue-ish header tint
   - In Review: amber/yellow header tint
   - Done: green header tint
   - Use subtle bg colors that work with the Ledger amber theme (use oklch-based classes or inline styles with low opacity)

5. **Data:** Import `tasks`, `getTasksByStatus`, `getUserById` from `@/data/ledger`. Group tasks by status. The `TaskStatus` type is: `"backlog" | "in-progress" | "in-review" | "done"`.

**Notes:**
- Server component (static layout, no drag-and-drop)
- On smaller screens, use horizontal scroll for the columns (don't stack them vertically вҖ” Kanban boards need horizontal layout to make visual sense)
- Add `overflow-x-auto` on the container for responsive behavior
- Each column should have a minimum width (~280px) to prevent cards from being too narrow
