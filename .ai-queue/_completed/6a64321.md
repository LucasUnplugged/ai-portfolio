# Segment 4: Integrate SiteHeader & SiteFooter into Root Layout

## Objective
Add the SiteHeader and SiteFooter to the root layout so they appear on all main site pages (home, case studies). Demo app pages have their own DemoHeader, so the site header/footer should NOT appear inside demo layouts.

## Scope
- `src/app/layout.tsx` (modify)
- `src/app/app/layout.tsx` (new — wrapper to exclude SiteHeader/SiteFooter from demos)

## Instructions

### Step 1: Create demo app group layout

Create `src/app/app/layout.tsx`:

This layout wraps all demo routes (`/app/ledger`, `/app/ritual`, `/app/circles`). Its purpose is to provide a clean boundary — demo routes use their own DemoHeader, NOT the SiteHeader/SiteFooter. This layout is a simple passthrough.

```tsx
import type { ReactNode } from "react";

export default function DemoAppLayout({ children }: { children: ReactNode }) {
  return <>{children}</>;
}
```

### Step 2: Update root layout

Modify `src/app/layout.tsx`:

- Import and add `SiteHeader` and `SiteFooter`
- Wrap `{children}` in a flex column layout so the footer is pushed to the bottom
- Keep all existing font imports and metadata unchanged

The updated body content should be:

```tsx
<body className={`${geistSans.variable} ${geistMono.variable} ${manrope.variable} ${bricolageGrotesque.variable} ${adlamDisplay.variable} ${afacad.variable} ${adamina.variable} ${firaCode.variable} ${inter.variable} ${jetbrainsMono.variable} antialiased`}>
  <div className="flex min-h-screen flex-col">
    <SiteHeader />
    <main className="flex-1">
      {children}
    </main>
    <SiteFooter />
  </div>
</body>
```

**Important:** Add these imports at the top:
```tsx
import { SiteHeader } from "@/components/site-header";
import { SiteFooter } from "@/components/site-footer";
```

**Note about demo pages:** The SiteHeader and SiteFooter will render on demo pages too since they're in the root layout. However, demo layouts already have their own themed wrapper with DemoHeader. The site header/footer will appear above/below the demo's themed area. This is acceptable for now — the site header gives users a way to navigate back, and the footer provides consistency. If the stacking looks wrong, the demo app group layout (`src/app/app/layout.tsx`) can be updated later to hide them.

### Step 3: Verify build

```bash
npm run build
```

All routes should build successfully:
- `/` (home) — has SiteHeader + SiteFooter
- `/case-studies/*` — has SiteHeader + SiteFooter
- `/app/*` — has SiteHeader + SiteFooter (above) + DemoHeader (inside themed area)

## Verification
- Root layout includes SiteHeader and SiteFooter
- `src/app/app/layout.tsx` exists as a group layout for demo routes
- All routes build successfully
- `npm run build` succeeds
