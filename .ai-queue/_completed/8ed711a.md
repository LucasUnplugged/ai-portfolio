# Ritual Shared Components & Bottom Navigation

Build the reusable Ritual components, bottom navigation, and RitualShell wrapper.

## Context

Tech spec: `.ai-epics/docs/ritual-demo/tech-spec.md`
This segment depends on segment 001 (mock data) being complete, since components reference the data types.

The Ritual demo is a dark-themed mobile app rendered inside the PhoneSimulator. All components should work well within ~375px width and use the Ritual theme (purple/fuchsia dark palette). The theme is applied via `.ritual-theme` CSS class which sets CSS custom properties — components just use `bg-background`, `text-foreground`, `text-primary`, etc.

Reference the Ledger components (`src/components/ledger/`) for pattern conventions (props interface, imports, etc.).

## Instructions

### 1. Bottom Navigation (`bottom-nav.tsx`)

Icon-only nav bar for the bottom of the phone screen.

Props:
```typescript
interface BottomNavProps {
  current: 'circle' | 'chat' | 'matches' | 'profile';
}
```

Requirements:
- 4 icons in a row: Users (circle), MessageCircle (chat), Heart (matches), User (profile)
- From lucide-react
- Active icon: `text-primary` (fuchsia). Inactive: `text-muted-foreground`
- Each icon is a Next.js `Link` to the corresponding route:
  - circle → `/app/ritual/circle`
  - chat → `/app/ritual/chat`
  - matches → `/app/ritual/matches`
  - profile → `/app/ritual/profile`
- Fixed height ~56px, `bg-card` background, `border-t border-border`
- Icon size: 20px (`h-5 w-5`)
- Use `flex items-center justify-around` layout
- No text labels (PRD requirement: icon-only)

### 2. RitualShell (`ritual-shell.tsx`)

Wraps page content with the bottom nav. Used by all screens except onboarding.

Props:
```typescript
interface RitualShellProps {
  current: 'circle' | 'chat' | 'matches' | 'profile';
  children: React.ReactNode;
}
```

Structure:
```tsx
<div className="flex flex-col h-full min-h-0">
  <div className="flex-1 overflow-y-auto">
    {children}
  </div>
  <BottomNav current={current} />
</div>
```

The key is that the bottom nav stays fixed at the bottom while content scrolls above it. The `h-full min-h-0` ensures it fills the phone simulator's content area.

### 3. Circle Member (`circle-member.tsx`)

Displays a member in the circle grid.

Props:
```typescript
interface CircleMemberProps {
  user: RitualUser;
  className?: string;
}
```

Requirements:
- Avatar with initials (use shadcn Avatar/AvatarFallback)
- Avatar has a `bg-secondary` background, `text-secondary-foreground`
- Name below avatar (truncated if long)
- Online status dot: green if online, yellow if away, hidden if offline
- Status dot positioned at bottom-right of avatar (absolute positioning)
- Compact layout — fits in a grid cell (~80px wide)
- The whole component is a link to `/app/ritual/profile`

### 4. Chat Bubble (`chat-bubble.tsx`)

Single chat message.

Props:
```typescript
interface ChatBubbleProps {
  message: ChatMessage;
  sender: RitualUser;
  isCurrentUser: boolean;
  isGroupChat?: boolean;
}
```

Requirements:
- Current user's messages: right-aligned, `bg-primary text-primary-foreground` (fuchsia bubble)
- Other users' messages: left-aligned, `bg-secondary text-secondary-foreground`
- Show sender name above bubble in group chat (not in DM)
- Show sender avatar on the left for other users' messages in group chat
- Timestamp below bubble (small, muted text, format as "2:34 PM")
- Prompt responses (`type === 'prompt-response'`): styled differently — add a subtle card-like border and a small label like "Response to prompt"
- Max width ~80% of container
- Rounded corners: `rounded-2xl`, with slightly different corner radius on the sender's side

### 5. Chat Input (`chat-input.tsx`)

Message input bar.

Props:
```typescript
interface ChatInputProps {
  placeholder?: string;
}
```

Requirements:
- Input field + send button in a row
- `bg-card` background, `border-t border-border`
- Input uses `bg-secondary` with `text-foreground`
- Send button: lucide-react `Send` icon, `text-primary` color
- Rounded input field
- This is purely visual — no actual send functionality needed
- Fixed at bottom of chat area (above bottom nav)

### 6. Prompt Card (`prompt-card.tsx`)

Daily prompt/icebreaker card.

Props:
```typescript
interface PromptCardProps {
  prompt: DailyPrompt;
  className?: string;
}
```

Requirements:
- Card with subtle border or `bg-card` background
- Icon at top (Sparkles from lucide-react, `text-primary`)
- Prompt text in serif font (`font-heading`) — this is the emotional moment
- Date below in muted text
- Response count if there are responses (e.g., "5 responses")
- Rounded corners, some padding

### 7. Match Card (`match-card.tsx`)

Match profile in the connections list.

Props:
```typescript
interface MatchCardProps {
  match: Match;
  user: RitualUser;
  className?: string;
}
```

Requirements:
- Avatar on the left
- Name + last activity or status text
- For `matched`: show "Matched" badge + link to DM
- For `pending-received`: show "Accept" and "Decline" buttons (visual only)
- For `pending-sent`: show "Pending" badge
- Compact row layout (like a contact list)

### 8. Profile Header (`profile-header.tsx`)

Profile display for the member profile screen.

Props:
```typescript
interface ProfileHeaderProps {
  user: RitualUser;
  className?: string;
}
```

Requirements:
- Large centered avatar (size "lg" or custom ~64px)
- Name in heading font (`font-heading text-xl`)
- Age + location below name
- Bio text
- Interests as small badges/tags (`bg-secondary text-secondary-foreground` pills)
- Centered layout
