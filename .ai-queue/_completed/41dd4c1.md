# Case Study Pages

## Objective
Build all 3 case study pages (Ledger, Ritual, Circles) with project descriptions, design decisions, and prominent demo launch buttons.

## Scope
- `src/app/case-studies/ledger/page.tsx` (modify)
- `src/app/case-studies/ritual/page.tsx` (modify)
- `src/app/case-studies/circles/page.tsx` (modify)

## Instructions

### Step 1: Build Ledger case study page

Replace `src/app/case-studies/ledger/page.tsx` with a full case study page.

**Content:**
- **Concept:** Multiplayer project management & knowledge base. Think Notion meets Linear — real-time collaboration with a warm, inviting aesthetic.
- **Goals:** Demonstrate desktop-first layout design, real-time collaborative patterns, complex data views (tables, kanban, timelines), and sophisticated information architecture.
- **Design Approach:** Warm amber palette with clean typography. Desktop-optimized layout using Manrope for UI and Bricolage Grotesque for headings. Emphasis on information density without feeling cluttered.
- **Prominent "Launch Demo" button** linking to `/app/ledger`
- **Tags:** Desktop, Collaborative, Real-time
- **Accent color:** amber

### Step 2: Build Ritual case study page

Replace `src/app/case-studies/ritual/page.tsx`.

**Content:**
- **Concept:** Structured, time-bound social connections. A mobile app for maintaining meaningful relationships through scheduled rituals — regular check-ins, shared activities, intentional togetherness.
- **Goals:** Demonstrate dark mode mobile design, expressive typography with serif fonts, time-based UI patterns, and emotional design that encourages connection.
- **Design Approach:** Deep purple/fuchsia palette on dark background. Uses Adamina (serif) for headings to convey warmth, Afacad for UI text, and Fira Code for timestamps. The dark theme creates intimacy.
- **Prominent "Launch Demo" button** linking to `/app/ritual`
- **Tags:** Mobile, Dark Mode, Social
- **Accent color:** purple/fuchsia

### Step 3: Build Circles case study page

Replace `src/app/case-studies/circles/page.tsx`.

**Content:**
- **Concept:** Relationship maintenance & personal CRM. A mobile app that helps you nurture your personal and professional relationships with gentle reminders and organized contact management.
- **Goals:** Demonstrate clean mobile design, minimal interface patterns, friendly micro-interactions, and data-light UI that focuses on people rather than numbers.
- **Design Approach:** Fresh teal/green palette on light background. Uses Inter for clean readability and JetBrains Mono for data elements. Rounded shapes and generous whitespace create a calm, approachable feel.
- **Prominent "Launch Demo" button** linking to `/app/circles`
- **Tags:** Mobile, Minimal, Personal
- **Accent color:** teal

### Design Guidelines (all 3 pages)

- Use a consistent layout template:
  1. Back link to home (the SiteHeader handles main nav, but a breadcrumb-style "← Back to Portfolio" link at the top of the content is helpful)
  2. Project name (h1) and one-liner
  3. Tags row
  4. "The Concept" section
  5. "Design Goals" section (as a list)
  6. "Design Approach" section
  7. Prominent CTA to launch the demo
- Use shadcn Card, Badge, Button, Separator
- Each page should have a subtle accent color matching the demo theme
- Responsive — single column on mobile, comfortable reading width on desktop
- Brief, sharp content — no walls of text. Each section 2-4 sentences max.
- The SiteHeader and SiteFooter come from the root layout, so don't add them
